Blogu GitHub Pages ile yayına almak

12 Temmuz 2026 4 dk okuma

Bu blog, build aracı gerektirmeyen statik dosyalardan oluşur. Bu yüzden GitHub Pages için hızlı ve dayanıklı bir yapı sunar.

Yayın mantığı

Ana sayfa yazı listesini gösterir. Her makale ise `posts/` klasöründe ayrı bir HTML dosyasıdır. Yeni yazı eklemek için mevcut yazılardan birini kopyalayıp içeriği değiştirmen yeterlidir.

Yayınlama adımları

  1. Yeni yazı dosyasını `posts/` klasörüne ekle.
  2. Ana sayfadaki makale listesine kartını koy.
  3. RSS için `feed.xml` dosyasına yeni kayıt ekle.
  4. Commit ve push sonrası GitHub Pages siteyi yayınlar.

Neden statik?

İçerik az kodla yönetilir, sayfalar hızlı açılır ve ücretsiz GitHub Pages barındırmasıyla kolayca yayında kalır.
